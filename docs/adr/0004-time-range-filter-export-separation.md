# ADR: 時間範囲フィルタと表示・エクスポートを分離する

- Status: Accepted
- Date: 2026-03-27

## Context

ログの時間範囲フィルタを追加する際、フィルタ結果を「表示」と「エクスポート」の両方に適用するか、分離するかを決める必要があった。

ユースケースとして「表示は特定時間帯に絞りたいが、エクスポートは全データを含めたい」という要件がある。例えば、問題のある時間帯を画面で確認しつつ、エクスポートには前後のコンテキストも含めたい場合。

また、UTC / ローカルタイム切り替え機能との整合性も考慮が必要だった。

## Decision

時間範囲フィルタは表示のみに適用し、エクスポートは「フィルタを適用するか全データを含めるか」をユーザーが選択できるようにする。

## Decision Details

- 時間範囲フィルタは表示テーブルの絞り込みに使用
- エクスポート時に「Export ignores time filter」トグルを提供
  - ON: 入力データの全時間範囲をエクスポート
  - OFF: 現在の時間範囲フィルタを適用してエクスポート
- 開始・終了時刻のデフォルト値は読み込んだファイルの時間範囲
- UTC / ローカルタイム切り替えは表示・フィルタ入力の両方に反映

## Alternatives Considered

### フィルタ結果をそのままエクスポート（分離なし）

不採用。全データエクスポートしたい場合にフィルタを一旦解除する必要があり、操作が煩雑。

### エクスポート専用の時間範囲を別途設ける

不採用。UI が複雑になりすぎる。トグル一つで切り替えられるほうがシンプル。

## Consequences

- 表示とエクスポートの時間範囲を独立に制御できる
- トグルの意味がわかりにくい可能性があるため、ツールチップで補足
- ノード・severity 等の他のフィルタはエクスポートに常に適用される（時間範囲のみが分離対象）

## Verification / Guardrails

- 時間範囲フィルタ適用後に表示件数が絞り込まれることを E2E テストで検証
- エクスポートトグル ON/OFF でエクスポート内容が変わることをテストで検証
- UTC/ローカル切り替え時にフィルタ入力値が正しく変換されることを検証
